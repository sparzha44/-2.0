О проекте Yatube

💡 Yatube — это социальная сеть для блогеров, построенная с использованием Django. Проект демонстрирует основные концепции веб-разработки: работу с БД, аутентификацию, пагинацию и работу с контекст-процессорами.

🏗️ Архитектура приложения

Приложение структурировано по принципу разделения ответственности:

  • posts — основная функциональность: создание постов, группы, поиск
  • users — управление аккаунтами, аутентификация, восстановление пароля
  • core — общие компоненты: контекст-процессоры, кастомные фильтры
  • about — статичные страницы информации о проекте

🎨 Ключевые особенности реализации

✨ Классовые представления (Class-Based Views)

Использование TemplateView, ListView и других CBV позволяет писать чистый и переиспользуемый код. Статичные страницы (Об авторе, Технологии) реализованы именно с помощью TemplateView.

🔐 Система аутентификации и авторизации

Использован встроенный Django User model. Для защиты view-функций применяется декоратор @login_required — это замыкание, которое проверяет аутентификацию пользователя перед выполнением функции.

Замыкание — это функция, которая "помнит" переменные из своей внешней области видимости. Декоратор и его внутренняя функция-обёртка образуют замыкание.

📄 Пагинация (Paginator)

Реализована постраничная навигация с показом 10 постов на странице. Django Paginator автоматически обрабатывает:

  • Разделение данных на страницы
  • Проверку существования страницы
  • Навигацию между страницами
🌐 Контекст-процессоры

Создана функция year() в core/context_processors/year.py, которая автоматически добавляет переменную {{ year }} во все шаблоны. Это позволяет избежать дублирования кода в каждой view-функции.

Контекст-процессор вызывается при каждом запросе, перед рендерингом шаблона.

🎯 Кастомные фильтры шаблонов

Реализован фильтр highlight для выделения найденных в поиске слов. Фильтр использует регулярные выражения для поиска и замены текста.

Техника: {% load highlight %} в шаблон, затем {{ text|highlight:"keyword"|safe }}

💾 Структура БД

Проект использует SQLite3 для разработки. Основные модели:

  • Post — запись в блоге (автор, текст, дата, группа)
  • Group — категория постов
  • User — встроенная модель Django для пользователей
